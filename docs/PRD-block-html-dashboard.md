# PRD: 블록 드래그 앤 드롭 HTML 생성 대시보드

**문서 버전:** 1.0  
**작성일:** 2025-03-13  
**기술 스택:** React, TypeScript, Vite, Tailwind CSS

---

## 1. 개요

### 1.1 목적

왼쪽 패널의 블록 아이콘을 오른쪽 캔버스로 드래그 앤 드롭하여 **테이블 기반 HTML**을 시각적으로 조합하고, 실시간 Preview와 생성된 HTML 코드를 탭으로 확인할 수 있는 대시보드를 제공한다.

### 1.2 핵심 가치

- **비개발자 친화적:** 블록 조합만으로 HTML 구조 생성
- **테이블 코딩:** 이메일/레거시 호환을 위한 테이블 레이아웃 HTML 출력
- **즉시 확인:** Preview / HTML 탭으로 결과를 바로 확인

### 1.3 대상 사용자

- 마케팅/운영 담당자 (이메일 HTML, 랜딩 페이지 초안)
- 프론트엔드 개발자 (테이블 레이아웃 HTML 빠른 프로토타이핑)

## 2. 기능 요구사항

### 2.1 전체 레이아웃

| 영역            | 위치 | 설명                                                    |
| --------------- | ---- | ------------------------------------------------------- |
| **블록 팔레트** | 좌측 | 드래그 가능한 블록 아이콘 목록 (고정 너비, 세로 스크롤) |
| **캔버스**      | 중앙 | 드롭 존; 블록이 쌓이는 작업 영역                        |
| **결과 패널**   | 우측 | 탭: Preview / HTML                                      |

```
┌─────────────┬──────────────────────────┬─────────────────────┐
│             │                          │  [Preview] [HTML]   │
│  블록 팔레트  │       캔버스 (드롭존)      │  ─────────────────  │
│             │                          │                     │
│  [이미지]    │   드롭된 블록들이         │   탭 콘텐츠 영역      │
│  [레이아웃]  │   여기에 쌓임             │   (미리보기 or 코드)  │
│  [텍스트]    │                          │                     │
│  [버튼]     │                          │                     │
│  ...        │                          │                     │
└─────────────┴──────────────────────────┴─────────────────────┘
```

### 2.2 좌측: 블록 팔레트

#### 2.2.1 블록 종류 (1차 범위)

| 블록 타입    | 아이콘/라벨              | 드롭 시 생성되는 HTML (테이블 코딩)                               |
| ------------ | ------------------------ | ----------------------------------------------------------------- |
| **이미지**   | 이미지 아이콘            | `<table><tr><td><img src="..." alt="..." /></td></tr></table>`    |
| **레이아웃** | 컬럼 아이콘 (1/2/3열 등) | 컬럼 수에 맞는 `<table><tr><td>...</td><td>...</td></tr></table>` |
| **텍스트**   | 텍스트 아이콘            | `<table><tr><td><p>텍스트</p></td></tr></table>`                  |
| **버튼**     | 버튼 아이콘              | `<table><tr><td><a href="#">버튼</a></td></tr></table>`           |
| **구분선**   | 수평선 아이콘            | `<table><tr><td><hr /></td></tr></table>`                         |
| **스페이서** | 간격 아이콘              | `<table><tr><td height="20"></td></tr></table>`                   |

- 각 블록은 **카드/블록 형태**로 표시 (아이콘 + 블록 이름).
- 블록은 **드래그 가능**하며, 드래그 중에는 반투명/썸네일로 피드백.

#### 2.2.2 블록 그룹 (선택)

- 그룹 예: **기본**, **레이아웃**, **미디어** 등으로 접을 수 있는 섹션으로 표시 가능.

### 2.3 중앙: 캔버스 (드롭 존)

- **드롭 존**: 블록을 놓을 수 있는 영역. 드래그 중일 때 드롭 가능 영역이 시각적으로 강조(테두리/배경 변경).
- **블록 순서**: 위에서 아래로 쌓임. (추후: 드래그로 순서 변경 가능)
- **각 블록 인스턴스**:
  - 선택 시: 테두리/핸들 표시, 삭제 버튼 등 (선택 사항)
  - 더블클릭 또는 패널에서: 이미지 URL, 텍스트 내용, 링크 등 **속성 편집** 가능 (2차 범위 권장)
- **초기 상태**: “여기에 블록을 드래그하세요” 플레이스홀더 메시지.

### 2.4 우측: 결과 패널 (탭)

#### 2.4.1 탭 구성

| 탭 이름     | 기능                                                                                        |
| ----------- | ------------------------------------------------------------------------------------------- |
| **Preview** | 캔버스에 쌓인 블록들을 조합한 **실제 HTML**을 iframe 또는 sandbox div에 렌더링하여 미리보기 |
| **HTML**    | 생성된 **전체 HTML 문자열** (테이블 코딩) 표시. 복사 버튼 제공 권장                         |

#### 2.4.2 HTML 출력 규칙 (테이블 코딩)

- **최상위**: `<table>` 로 감싼 구조.
- **블록 단위**: 각 블록이 `<tr>` 또는 내부 `<table>` 로 매핑되어, 시맨틱하지 않더라도 테이블 기반 레이아웃으로 출력.
- **일관된 인덴트**: HTML 탭에서는 가독성을 위해 포맷팅(들여쓰기) 적용 권장.

예시 출력 구조:

```html
<table role="presentation" width="100%" cellpadding="0" cellspacing="0">
  <tr>
    <td><!-- 이미지 블록 --></td>
  </tr>
  <tr>
    <td><!-- 텍스트 블록 --></td>
  </tr>
  <tr>
    <td><!-- 버튼 블록 --></td>
  </tr>
</table>
```

### 2.5 드래그 앤 드롭 동작

- **시작**: 블록 팔레트에서 블록을 마우스로 드래그.
- **이동 중**: 캔버스 위에서만 드롭 가능 커서/스타일 표시.
- **드롭**: 캔버스에 해당 블록 타입의 인스턴스 추가 → 데이터 모델 업데이트 → Preview/HTML 탭 자동 반영.
- **라이브러리**: HTML5 DnD API 또는 `@dnd-kit/core` 등 React 친화 라이브러리 사용 권장.

## 3. 비기능 요구사항

### 3.1 기술 스택

- **프레임워크:** React 19
- **언어:** TypeScript
- **빌드:** Vite 7
- **스타일:** Tailwind CSS 4
- **상태:** 컴포넌트 상태 + (필요 시) Context 또는 경량 상태 관리

### 3.2 호환성

- 최신 Chrome, Safari, Firefox, Edge (최근 2개 버전)
- 반응형: 1280px 이상 해상도 우선, 필요 시 1024px 이하에서 패널 비율 조정

- 블록 50개 수준에서 드래그/드롭/리렌더 지연 최소화
- Preview iframe 재렌더는 디바운스(예: 300ms) 적용 권장

### 3.4 접근성

- 키보드로 블록 포커스 및 탭 전환 가능 (선택)
- 시맨틱 마크업 및 aria 레이블 적용

## 4. 데이터 모델 (간이)

### 4.1 캔버스 블록 리스트

```ts
type BlockType = "image" | "layout" | "text" | "button" | "divider" | "spacer";

interface BlockBase {
  id: string;
  type: BlockType;
}

interface ImageBlock extends BlockBase {
  type: "image";
  src: string;
  alt?: string;
}

interface TextBlock extends BlockBase {
  type: "text";
  content: string;
}

// ... 기타 블록 타입

type CanvasBlock =
  | ImageBlock
  | TextBlock
  | ButtonBlock
  | LayoutBlock
  | DividerBlock
  | SpacerBlock;
```

- 캔버스 상태: `CanvasBlock[]`
- 블록 추가/삭제/순서 변경 시 이 배열만 업데이트하고, 동일 데이터로 Preview용 HTML과 HTML 탭용 문자열 생성.

### 4.2 HTML 생성

- `blocksToTableHtml(blocks: CanvasBlock[]): string` 형태의 순수 함수로 구현 권장.
- 블록 타입별로 테이블 코딩 스니펫 매핑.

### 6.4 의존성 관계

- **Phase 2**는 Phase 1의 캔버스·블록 모델이 완성된 후 진행.
- **Phase 3**의 저장/불러오기는 Phase 2의 블록 구조가 안정된 후 권장.
- 레이아웃 블록(2.1)은 Phase 1의 HTML 생성 로직 확장만으로 가능해 Phase 1 말기에 앞당길 수 있음.

## 7. 성공 지표

- 사용자가 블록을 드래그하여 5개 이내 블록으로 구성된 HTML을 1분 이내에 생성 가능
- 생성된 HTML이 테이블 기반으로 일관되게 출력됨
- Preview와 HTML 탭 전환이 즉시 반응

---

## 8. 제약 및 가정

- **테이블 코딩**: 이메일 클라이언트 호환 등 레거시 요구를 전제로 한 테이블 레이아웃만 1차 지원. 시맨틱 HTML/div 레이아웃은 2차 검토.
- **스타일**: 인라인 스타일 또는 `<style>` 주입은 2차 범위로 두고, 1차는 구조(테이블) 위주.
- **저장**: 1차는 세션 내 상태만 유지해도 가능; 저장/불러오기는 Phase 3.
